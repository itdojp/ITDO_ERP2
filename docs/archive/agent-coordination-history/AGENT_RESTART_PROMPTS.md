## 🎯 最適化された再開プロンプト

### 1. 汎用再開プロンプト（推奨）
```

```

**利点**: 状況確認・指示作成・問題解決を網羅

### 2. 最小プロンプト
```

```
**トークン数**: 約15
**利点**: 最小トークンで基本機能実行


**トークン数**: 約25
**利点**: 問題解決に焦点

### 4. 進捗確認型
```
CC01,02,03の進捗確認。完了タスクには新フェーズを割り当ててください。
```
**トークン数**: 約20
**利点**: 継続的な作業割り当て

### 5. 緊急対応型
```
CC01,02,03緊急確認。24時間以上停止なら復活プロトコル実行。
```
**トークン数**: 約20
**利点**: 長期停止の自動検出と対応

## 🚀 プロンプト最適化のポイント

### 1. 必須要素
- エージェント名明記（CC01,02,03）
- 行動指示（確認、作成、支援）
- 規約参照（CLAUDE.md準拠）

### 2. 効率化テクニック
- 番号での指定: CC01,02,03
- 動詞の簡潔化: 確認→確認と指示
- 条件付き指示: 問題があれば支援

### 3. 避けるべき表現
- 冗長な説明
- 重複する指示
- 曖昧な要求
